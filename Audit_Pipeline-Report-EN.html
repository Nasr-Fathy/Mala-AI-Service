1. Architecture Overview & Upgrades

The project pipeline consists of two main stages: 1. OCR Capture and 2. Financial Mapper.
In the legacy version, all tasks relied on a single monolithic AI model, leading to severe bottlenecks in cost, execution time, cumulative errors, and exceeding model thinking limits. The architecture has been completely re-engineered to decouple tasks, utilize specific models based on task complexity, and implement parallel processing.

3. Major Challenges Resolved

  • Over-thinking Issue: The old model frequently over-thought tasks, exceeding 60,000 tokens per task, which caused massive cost spikes and fallbacks.
    Solution: Implemented strict controls over the "Thinking Level" and "Thinking Budget", completely eliminating this issue.
  • Cumulative Errors & Fallbacks: Due to the weak old model and incorrect schemas, requests were retried up to 5 times, multiplying time and cost exponentially.
    Solution: Upgrading the OCR stage to the Pro model stabilized the schema generation, reducing the fallback rate to nearly 0%.
  • Sequential Execution Bottleneck: The system used to wait for each pass to finish before starting the next.
    Solution: Decoupled Pass 1 and Pass 2 to run in parallel. The execution time is now equal to the longest task, not the sum of both.
  • Error Handling & Monitoring: The legacy system operated as a "black box".
    Solution: LangSmith integration now provides granular visibility into every step, resulting in vastly improved error handling and better overall management.

4. Technical Conclusion

Transitioning from a "Single Sequential Model" to a "Distributed, Parallel, Task-Specific" architecture achieved an optimal balance between performance quality and operational cost. Initial testing on 6 files demonstrated 100% accuracy, with the entire pipeline completing in approximately 2 minutes. This upgrade drastically reduced costs, resolved token over-consumption, and established robust observability.
